## 后续使用原则

每次交付前，至少检查：

- 是否使用了真实 Figma 节点数据和真实素材。
- 是否存在未说明的 CSS 手绘、近似替代、字体替代或视觉验收缺口。
- 是否已经把本次发现的新问题写进审计脚本或经验文档。
- 是否完成浏览器实际检查，而不只依赖文件存在或脚本通过。

规则冲突时按这个顺序判断：

1. Figma 真实数据。
2. 真实素材来源。
3. 浏览器最终视觉。
4. 局部视觉验收。
5. 审计脚本。
6. CSS 实现便利。
