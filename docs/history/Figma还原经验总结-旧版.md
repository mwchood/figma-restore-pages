# Figma 还原规则手册

这份文档用于后续所有 Figma 页面还原。目标不是“看起来差不多”，而是尽量做到可追溯、可审计、可解释的高保真还原。

## 一、最高原则

当规则之间出现冲突时，按下面优先级判断：

0. **项目 MD 规则优先读取**
   - 每次开始新的 Figma 页面还原前，必须先读取本项目的还原规则 MD 文档，再读取 Figma 数据和实现页面。
   - 不能只凭上下文记忆执行规则；如果规则文件缺失或读取失败，必须先说明风险。

1. **Figma 真实数据优先**
   - 以 Figma metadata、节点 bounds、图层结构、文本参数、真实素材来源为第一依据。
   - 不能只凭截图观感猜坐标、猜尺寸、猜图标。

2. **真实素材优先**
   - 图标、图片、背景、纹理、渐变、遮罩、复杂矢量必须优先来自 Figma 真实节点或真实素材路径。
   - 没拿到真实素材时，不允许悄悄用 CSS 手绘或自制 SVG 替代。

3. **浏览器最终视觉优先**
   - 素材真实、节点正确、脚本通过，都不能覆盖浏览器里肉眼可见的不一致。
   - 如果浏览器截图和 Figma 不一致，要继续排查，不能用“素材已经加载”作为完成理由。

4. **局部视觉验收优先于全局脚本通过**
   - 对组合图表、图标组、商品图、背景纹理、渐变卡片等高风险区域，必须做局部对比。
   - 没做局部对比时，只能说“结构/素材检查通过”，不能说“视觉完全还原”。

5. **审计脚本用于防复发**
   - 用户指出过的问题，必须写进脚本或明确写进交付限制。
   - 脚本通过只是必要条件，不是充分条件。

6. **CSS 实现便利最低优先级**
   - 不能因为 CSS 好写，就牺牲 Figma 数据、真实素材、视觉一致性。

## 二、冲突处理规则

### 1. Figma bounds 与 SVG natural size 冲突

优先判断 SVG 是哪种导出：

- 如果 SVG 是完整节点导出，包含原始 padding 和 viewBox，按 Figma bounds 放置。
- 如果 SVG 是紧裁剪 painted bbox，不能强行拉伸到 Figma 外框，否则会变形。
- 如果需要完整节点视觉，应重新导出带 padding 的完整节点，而不是拉伸紧裁剪 SVG。

示例：

- Typhur Wellness 圆弧 `66:8223` 导出为 `176x161`，`66:8224` 导出为 `173x161`。
- 强行设为 `176x176` 会让圆弧纵向拉伸，端点超出轨道。

### 2. “完整图标节点”与“不要导出外层 frame”冲突

正确做法：

- 使用设计稿中可见的语义节点，不能偷用更小的内部 path。
- 但 SVG 里如果带了背景 rect、页面 rect、调试底色，要剥离这些包装层。
- 剥离背景不等于换成手写路径，也不等于改图标形状。

必须检查：

- 是否有 `fill="#F5F5F5"` 这类包装灰底。
- 是否有页面背景 rect。
- 是否有多余的父级 frame。
- 是否保留了真实图标 path。

### 3. 局部导出素材与禁止截图还原的边界

允许：

- 导出 Figma 局部节点资产。
- 导出真实照片、产品图、复杂背景、纹理、遮罩、渐变卡片、复杂矢量组。

禁止：

- 用整页截图当页面实现。
- 用大块截图逃避文本、布局、控件还原。
- 把截图当作“真实素材”来掩盖没有读 Figma 图层的问题。

### 4. 字体真实度与无版权风险方案

如果没有授权字体：

- 可以使用系统或开源替代字体。
- 必须说明不是 1:1 字形还原。
- 仍要按 Figma 对齐字号、字重、行高、颜色、位置。

不能说：

- “字体完全一致”
- “1:1 字体还原”

除非真实字体文件已经加载并通过浏览器检查。

### 5. 脚本通过与视觉通过冲突

如果脚本通过但用户或截图指出视觉问题：

- 以视觉问题为准。
- 找出脚本为什么没拦住。
- 把这类失败模式加入审计。

典型例子：

- `failedImages: []` 只能说明图片加载了，不能说明图表比例正确。
- 素材路径真实，不代表 CSS 尺寸正确。
- SVG 来自 Figma，不代表里面没有灰底或背景 rect。

## 三、素材规则

### P0 必须遵守

- 不允许悄悄手绘图标。
- 不允许用 CSS 猜复杂背景、纹理、渐变、遮罩。
- 不允许用整页截图当实现。
- 不允许把整页截图或局部截图裁成小图标/小素材来伪装成“真实素材”；除非用户明确批准临时近似方案，否则宁可标记素材缺失。
- 不允许拿不到素材时直接降级。
- 不允许把临时替代伪装成真实还原。

### P1 素材获取顺序

1. 读取 Figma metadata 和 design context。
2. 找到真实节点 id。
3. 导出真实节点 SVG/PNG/JPEG。
4. 下载到本地 `assets/`。
5. 检查素材内容是否夹带背景、父级 frame、错误尺寸。
6. 在页面中引用本地真实素材。

### P1 图标规则

- 图标优先 SVG。
- 图标不能使用 PNG 截图替代，避免模糊。
- SVG 不能有意外灰底、白底、页面背景 rect。
- 线性图标要检查 `fill`，避免默认黑色填充。
- 布尔/填充图标不能手工简化 path。

### P1 图片规则

- 产品图不能拉满整个卡片，必须按 Figma 子图层尺寸放置。
- 商品卡片要先还原背景块，再放产品图。
- 图片自带白底时，要确认 Figma 里是否也可见白底。
- 如果 Figma 里没有白底，需使用透明导出、混合模式或正确子节点素材。

## 四、布局规则

### P0 坐标规则

- 页面根元素使用 Figma 页面坐标。
- 子元素必须使用父级相对坐标。
- 不能把 Figma 页面绝对坐标直接写进卡片内部 CSS。

示例：

- Typhur Wellness 文字框 `66:8225` 页面 x 为 `90`。
- 统计卡片 x 为 `16`。
- 卡片内 CSS 应为 `left: 74px`，不是 `90px`。

### P1 图层规则

- 保留重要元素的 `data-figma-id`。
- 文字节点 id 放在真实文字元素上。
- 背景、卡片、文本、图标不要混成一个不可检查的大图。
- Figma 隐藏层不能随意启用。

### P1 CSS 选择器规则

- 不写过宽泛选择器。
- 大图规则要限定直接子元素，例如 `.food-card > img`。
- 小图标单独写尺寸。
- 全局规则可能影响局部布局，必须检查覆盖影响。

## 五、视觉规则

### P0 高风险区域必须局部验收

以下区域不能只靠脚本和素材加载判断：

- 统计环、进度环、仪表盘、营养图表。
- 多层图标组、tabbar、状态栏。
- 商品图、食品图、卡片背景。
- 渐变、纹理、光效、遮罩、玻璃效果。
- Home Indicator、底部按钮区。
- 数字与单位组合，例如时间选择器、价格、温度、重量、百分比、倒计时。

必须检查：

- 圆心。
- 比例。
- 弧线长度。
- 端点位置。
- 点阵半径。
- 文字字号和字重。
- 数字与单位是否在同一视觉基线；不能只看 `x/y/字号`，尤其不能让 `hr/min/°C/¥/%` 这类单位因为独立绝对定位而下沉或上浮。
- 图层对齐。
- 是否有裁切、拉伸、灰底、白底、模糊。

### P1 组合件规则

如果一个组件由多个 Figma 子节点拼装，必须记录：

- 每个子节点 id。
- 每个本地素材路径。
- 每个素材的 natural size。
- CSS 显示尺寸。
- 是否做过局部截图对比。

如果没做局部截图对比，必须标记：

> 该组合件尚未完成局部视觉验收。

## 六、字体规则

### P0 字体声明规则

- 没有真实字体文件时，不能声称字体 1:1。
- 使用替代字体时，要说明可能存在字宽、字面高度、粗细观感差异。

### P1 检查项

- font-family。
- font-size。
- font-weight。
- line-height。
- color。
- letter-spacing。
- 文本框宽高。
- 是否溢出。

## 七、审计规则

### P0 审计必须覆盖用户指出过的问题

每次用户指出问题后，要问：

- 这是什么类型的问题？
- 为什么脚本没发现？
- 能不能写成自动检查？
- 如果不能自动检查，是否写进交付限制？

### P1 当前必须拦截的问题

- 真实素材路径缺失。
- 使用临时 MCP URL。
- 手绘图标回归。
- SVG 背景 rect 回归。
- 小图标被大图 CSS 拉伸。
- 紧裁剪 SVG 被拉伸变形。
- 父级相对坐标写成页面绝对坐标。
- 文字字号、行高、宽度导致溢出。
- 数字+单位组合的基线或垂直对齐漂移，例如 `03 hr`、`30 min` 这类小面积问题不能被整页 diff 稀释掉。
- Home Indicator 坐标复用错误。
- 图片白底/灰底未处理。

## 八、交付前检查清单

交付前必须完成：

- 读取 Figma metadata。
- 读取 design context。
- 明确真实素材来源。
- 本地化关键素材。
- 检查 SVG 内容。
- 检查浏览器图片加载。
- 检查关键元素实际渲染尺寸。
- 检查文字是否溢出。
- 对高风险区域做局部视觉检查。
- 运行审计脚本。
- 说明未完成的视觉验收项。

## 九、如果做不到

如果缺少工具、权限、字体、素材或视觉验收能力，必须直接告诉用户：

- 缺什么。
- 为什么影响还原。
- 会留下什么风险。
- 用户可以提供什么。
- 是否需要允许临时近似。

不能继续假装已经完成。

## 十、一句话规则

真实素材不等于视觉正确，脚本通过不等于还原完成。
Figma 数据、真实素材、浏览器视觉、审计脚本必须同时闭环，才能交付。
