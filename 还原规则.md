# Figma 页面还原规则

这是本项目唯一有效的规则入口。开始任何 Figma 页面还原、修复、审计或发布前，必须先读取本文件和 [还原流水线.md](./还原流水线.md)。

## 1. 优先级

发生冲突时按以下顺序处理：

1. **P0 真实性与不降级**：真实 Figma 数据、真实素材、真实限制必须如实呈现。
2. **P0 最终视觉结果**：以目标视口中浏览器实际渲染结果为准。
3. **P0 审计覆盖不得回退**：文档整理、脚本重构或流水线合并不能删除既有检查能力。
4. **P1 结构与可维护性**：保留节点追溯、父子关系和项目现有实现方式。
5. **P2 自动审计结果**：脚本通过是交付条件之一，但不能替代视觉验收。
6. 历史档案仅用于追溯经验，不能覆盖当前规则。

## 2. P0 真实性与不降级

### 禁止事项

- 禁止把整页截图作为页面实现。
- 禁止裁切截图并伪装成图标、图片或“真实素材”。
- 禁止未经用户批准，用 CSS、手绘 SVG、图标库、占位图或近似素材替代 Figma 中应存在的真实素材。
- 禁止使用会过期的 Figma MCP 临时地址交付；关键素材必须本地化到 `assets/`。
- 禁止在素材、字体、权限或视觉工具缺失时悄悄降级。
- 禁止只凭截图猜测坐标、尺寸、字重、图层和素材来源。
- 禁止把文件存在、图片加载成功、脚本通过或线上返回 200 描述成“视觉还原完成”。

### 无法完成时

继续实现前必须说明：

- 缺少的数据、素材、字体、权限或工具。
- 缺失项为什么影响还原。
- 会留下哪些视觉风险。
- 用户可以提供什么。
- 是否需要用户明确批准临时近似方案。

未经批准，不执行临时近似方案。

## 3. P0 审计覆盖不得回退

### 合并与重构守恒

- 精简文档只能减少入口，不能减少检查项。
- 旧规则或用户指出过的问题必须迁移到以下至少一处：自动流水线、页面 manifest、专项审计或强制人工视觉区域。
- 删除、归档或合并审计文档前，必须建立“旧检查项到新落点”的对应关系。
- 没有新落点的检查项不得归档。
- 修改流水线后，必须用一个已知失败案例证明新流水线仍能拦截该类问题。

### 用户反馈闭环

每次用户指出漏检后必须回答并落实：

1. 这属于哪一类根因。
2. 为什么现有审计没有发现。
3. 能否转成自动检查。
4. 如果不能自动检查，必须加入哪个局部视觉验收项。
5. 使用当前失败页面验证新规则确实会失败，再修复并验证通过。

禁止只把问题追加到文档而不改变验收方式。

## 4. P1 数据与素材

- 必须获取目标节点的 design context、metadata 和 Figma 截图。
- 图标优先使用对应 Figma 节点导出的 SVG。
- 照片和产品图使用 Figma 原始图片或节点导出，不能从页面截图裁取。
- 渐变、纹理、光效、遮罩和复杂卡片优先使用真实节点导出，不能凭感觉用 CSS 猜。
- 必须记录关键素材的节点 ID、来源和 natural size。
- 必须检查 SVG 是否夹带灰底、白底、页面背景、父级 Frame 或错误填充。
- 完整节点 SVG 与紧裁剪 SVG 必须区分，不能把紧裁剪素材拉伸到父级 bounds。
- CSS 选择器不能宽泛到影响组件内部的小图标。

CSS 可以实现 Figma 中本身就是基础几何或布局属性的内容，例如纯色矩形、边框、圆角和标准排版；不能用它替代应当导出的设计素材。

## 5. P1 布局与关系

- 页面根元素使用 Figma Frame 尺寸和渲染比例。
- 子元素坐标必须换算为父级相对坐标。
- 除绝对坐标外，必须保留 Figma 的布局语义：居中、基线、约束、间距、裁切和层叠关系。
- 固定画布、图表、圆环、网格、按钮、图标和组合数字必须设置稳定尺寸。
- 不只检查单个元素的 `top/left/width/height`，还要检查关键元素之间的关系。

以下关系必须优先写成浏览器运行时审计：

- 文字与按钮、标签或色块的水平和垂直中心。
- 图标与文字的基线、间距和整体组合中心。
- 数字与单位的基线和视觉间距。
- 子元素是否完整包含在父元素中。
- 相邻行、分割线、选中态和背景块的相对位置。
- 经过缩放的 Frame 中，所有关键坐标是否使用同一比例。

## 6. P1 字体与文本

- 必须记录 font family、size、weight、line height、color、letter spacing、文本框尺寸和对齐方式。
- 没有授权字体文件时，使用无版权风险的系统或开源近似字体，并说明无法保证字形 1:1。
- 替代字体加载后必须重新检查换行、基线、视觉中心、宽度和溢出，不能沿用原字体坐标后直接交付。
- 浏览器运行时必须检查关键文字是否被裁切、溢出、错误换行或遮挡其他内容。
- CSS 几何中心不等于字形视觉中心；高风险组合仍需局部截图确认。

## 7. 四层审计模型

每个页面必须依次通过四层审计，后层不能被前层替代。

### A. 来源审计

- 节点、Frame、素材、字体与截图来源真实。
- 无临时 MCP URL、截图裁切或未批准替代方案。

### B. 静态结构审计

- HTML 中保留关键 `data-figma-id`。
- manifest、素材、natural size、关键 CSS 参数完整。
- 无错误背景包装、拉伸、翻转或过宽选择器。

### C. 浏览器运行时审计

- 检查根画布位置、尺寸、缩放、溢出和图片加载。
- 检查关键元素实际 bounds。
- 检查中心、边缘、包含、间距和基线等相对关系。
- 检查关键文本的实际适配情况。

### D. 视觉审计

- 生成与 Figma 相同视口的浏览器截图。
- 对高风险区域单独放大对比，不能只看整页缩略图。
- 视觉审计必须记录检查区域和结论；“看过整页”不算完成。
- 字体替代、抗锯齿和光效等无法可靠自动判断的内容，以局部视觉结果为准。

## 8. 强制局部视觉区域

以下内容必须单独截图或放大检查：

- 状态栏、导航栏、tabbar、组合图标和 Home Indicator。
- 圆环、仪表盘、图表、曲线、点阵和复杂矢量。
- 产品图、食物图、渐变、纹理、遮罩、光效和复杂背景。
- 底部操作区、固定按钮和安全区。
- 数字与单位、时间滚轮、价格、温度、百分比和倒计时。
- 文字位于按钮、色块、标签或卡片中的区域。
- 使用替代字体且位置敏感的标题、说明和多行文字。
- 用户曾指出过问题的同类区域。

## 9. 交付用语

- **结构审计通过**：只表示来源、文件和基础参数通过。
- **运行时审计通过**：表示真实浏览器中的尺寸、关系和适配通过。
- **视觉审计通过**：表示已完成规定的局部对比。
- **发布完成**：以上审计均通过，且线上页面与关键素材验证成功。

不得把较低等级的通过描述成较高等级完成。

## 10. 文档维护

- 通用原则和优先级只写入本文件。
- 操作顺序、manifest 字段和命令只写入 `还原流水线.md`。
- 页面参数写入 `restore-manifests/`。
- 可自动执行的规则必须进入流水线或专项脚本，不能只停留在文字中。
- 历史记录统一放入 `docs/history/`，保持只读追溯状态。
