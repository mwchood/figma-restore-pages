# Figma 页面还原规则

这是本项目唯一有效的规则入口。开始任何 Figma 页面还原、修复或审计前必须先读取本文件。

## 优先级

规则冲突时按以下顺序处理：

1. Figma 真实结构、节点参数和素材来源。
2. 浏览器中的最终视觉结果。
3. 本文件规定的素材、布局和验收要求。
4. 页面 manifest 与自动审计结果。
5. 历史档案只能用于追溯，不能覆盖当前规则。

## P0 禁止事项

- 禁止把整页截图作为页面实现。
- 禁止裁切截图并伪装成图标、图片或“真实素材”。
- 禁止在未告知用户的情况下用 CSS、手绘 SVG、图标库或占位素材替代 Figma 真实素材。
- 禁止使用会过期的 Figma MCP 临时地址交付，关键素材必须下载到本地 `assets/`。
- 禁止素材获取失败、字体缺失或视觉工具不可用时悄悄降级。
- 禁止只凭截图猜测坐标、尺寸、字重、图标和素材。
- 禁止把脚本通过、图片加载成功或路径真实当成视觉还原完成。

## 必须执行的顺序

1. 读取本文件和 [还原流水线.md](./还原流水线.md)。
2. 使用 Figma MCP 获取目标节点的 design context。
3. 获取 metadata 和 Figma 截图，确认 Frame、父子层级和坐标。
4. 找到每个图标、图片、背景、渐变、纹理和复杂矢量的真实节点。
5. 将关键素材下载到 `assets/`，记录节点 ID 和 natural size。
6. 检查 SVG 是否夹带灰底、白底、页面背景、父级 Frame 或错误填充。
7. 按项目现有 HTML/CSS 结构实现页面，保留关键 `data-figma-id`。
8. 建立或更新 `restore-manifests/*.json`。
9. 运行统一流水线和专项审计。
10. 对高风险区域进行浏览器截图对比。
11. 通过后加入统一菜单、提交 GitHub，并验证线上 HTML 与素材。

## 素材规则

- 图标优先使用 Figma 节点导出的 SVG。
- 照片和产品图使用 Figma 原始图片或节点导出，不能从页面截图裁取。
- 渐变、纹理、光效、遮罩和复杂卡片优先导出真实节点，不能凭感觉用 CSS 猜。
- 完整节点 SVG 与紧裁剪 SVG 必须区分，不能把紧裁剪素材强行拉伸到父级 bounds。
- SVG 中的背景包装层可以剥离，但必须保留真实图标或图形路径。
- 不允许使用过宽的 CSS 图片选择器影响内部小图标。

## 布局规则

- 页面根元素使用 Figma Frame 坐标。
- 子元素必须换算为父级相对坐标，不能直接复用页面绝对坐标。
- 固定画布、图表、圆环、网格、按钮和图标必须设置稳定尺寸。
- 检查实际浏览器中的元素 bounds，而不只检查 CSS 源码。
- 检查文字、圆弧、图标和图片是否溢出、裁切、拉伸、翻转或产生意外底色。

## 字体规则

- 必须记录 font family、size、weight、line height、color、letter spacing 和文本框尺寸。
- 没有真实授权字体文件时，使用无版权风险的系统或开源近似字体。
- 使用替代字体时可以做有限视觉补偿，但必须说明无法保证字形 1:1。
- 禁止在真实字体未加载时声称字体完全一致。

## 高风险区域

以下区域必须做局部截图对比：

- 圆环、仪表盘、图表、曲线和点阵。
- 状态栏、导航栏、tabbar 和组合图标。
- 产品图、食物图、渐变、纹理、光效和复杂背景。
- Home Indicator 和底部操作区。
- 数字与单位组合，如温度、时间、价格、百分比和倒计时。

## 无法完成时

必须在继续实现前告诉用户：

- 缺少什么数据、素材、字体、权限或工具。
- 为什么会影响还原。
- 会留下什么视觉风险。
- 用户可以提供什么。
- 是否需要用户明确批准临时近似方案。

未经用户批准，不执行临时近似方案。

## 文档维护

- 新的通用规则只写入本文件。
- 流程和命令只写入 `还原流水线.md`。
- 页面级参数写入 `restore-manifests/`，不再新增页面审计入口文档。
- 旧记录统一放入 `docs/history/`，并保持只读历史状态。
